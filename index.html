Introduction

Explore the differences between accessible and inaccessible web design. Use the examples below to experience the challenges faced by users with disabilities.

Focus Management: Accessible vs. Inaccessible

Proper focus management ensures users can navigate your website effectively with a keyboard. This includes logical focus order, visible focus indicators, and keyboard operability.

Accessible Focus Management

This navigation has a logical tab order, clear focus outlines, and is fully operable via keyboard.

  • Home
  • About
  • Services
  • Contact

Inaccessible Focus Management

This navigation disrupts the logical tab order, lacks focus indicators, and uses non-focusable elements.

Home About Services Contact

Pointer Interactions: Accessible vs. Inaccessible

Pointer interactions should allow users to cancel actions and ensure targets are large enough to be easily activated.

Accessible Pointer Cancellation and Target Size

This button allows users to cancel an action by moving the pointer away before releasing it, and the target size is at least 44x44 pixels.

Try it: Click and hold the button, then move the pointer away before releasing. The action will not trigger.

Inaccessible Pointer Cancellation and Target Size

This button triggers an action immediately on press, does not allow cancellation, and has a small target size.

Try it: Click and hold the button. Notice that the action triggers immediately and cannot be canceled.

Concurrent Input Mechanisms: Accessible vs. Inaccessible

Content should support multiple input methods, such as keyboard, mouse, and touch.

Accessible Concurrent Input Mechanisms

This slider works with both keyboard and touch inputs.

Slider Value: 50

Try it: Use the arrow keys or drag the slider to adjust the value.

Inaccessible Input Mechanisms

This slider only works with touch input and is not operable via keyboard.

Try it: Attempt to use the keyboard to interact with the slider. Notice it does not work.

Animation from Interactions: Accessible vs. Inaccessible

Animations triggered by user interactions should either not occur or be easily disabled.

Accessible Animation Control

This animation can be toggled on and off by the user.

Inaccessible Animation

This animation occurs automatically upon interaction and cannot be disabled.

Visual Design: Accessible vs. Inaccessible

Sufficient color contrast, text alternatives, and reflow ensure content is perceivable for all users.

Accessible Visual Design

This example provides sufficient contrast, descriptive alt text, and zoom-friendly layout.

This text has sufficient contrast.

Zoom in on this text and notice how it adjusts without breaking the layout.

Inaccessible Visual Design

This example lacks alt text, has poor contrast, and breaks when zoomed in.

This text has poor contrast.

Zoom in on this text and notice how it overflows or becomes unreadable.
